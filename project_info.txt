## Project 77 — WeatherFetch

**Tagline:** CLI app to fetch weather using a public REST API.

**Category:** Networking / API

### Problem Statement
Understanding how to communicate with external web services is a core developer skill. WeatherFetch demonstrates HTTP GET requests.

### Detailed Description
WeatherFetch connects to a free public API (like OpenWeatherMap) to fetch real-time weather data for a given city. It parses the JSON response and prints a beautifully formatted summary to the console, including temperature, humidity, and conditions. It handles network timeouts and invalid city names gracefully.

### Core Features
- Fetch current weather by city name
- Parse JSON API responses
- Display formatted output
- Handle HTTP error codes (404, 500)

### Technical Challenges
- Managing API keys securely (environment variables)
- Handling JSON deserialization dynamically
- Dealing with network latency or dropped connections

### Suggested Tech Stack
- Java 21 (java.net.http.HttpClient)
- Jackson/Gson
- OpenWeatherMap API

### Recommended Architecture
[CLI] -> [WeatherService] -> [HTTPClient] -> [External API]

### Java Concepts Demonstrated
- REST APIs
- HTTP Client
- JSON Parsing
- Exception Handling
